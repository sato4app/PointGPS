## 1. プロジェクト概要

### 1.1 プロジェクト名
**PointGPS - ポイント(GPS)編集アプリ**

### 1.2 目的
国土地理院地図上でGPSポイントの表示、編集、管理を行うWebアプリケーション。Excel専用化により30-50倍の大幅高速化を実現し、厳密なデータ検証機能とフィールド配置の最適化を提供。

### 1.3 主な用途
- ハイキング・登山トレイルのポイント管理
- GPS座標データの可視化・編集
- ExcelフォーマットでのGPSデータの入出力
- 地図上でのポイント配置の視覚的確認

### 1.4 対象ユーザー
- ハイカー、登山者
- GPS データを扱う地理情報管理者
- 地図アプリケーション開発者

## 2. 主要機能

### 2.1 データ読み込み機能

#### 2.1.1 Excel専用読み込み
- **対応フォーマット**: Excel (.xlsx) のみ
- **読み込みボタン**: 「ポイント読み込み(Excel)」
- **処理速度**: 100ポイント約300-500ms（30-50倍高速化達成）
- **行数制限**: 最大1000行まで（設定可能: `CONFIG.MAX_EXCEL_ROWS`）

#### 2.1.2 必須列構成
以下の列が完全一致で必須：
- **区分**: ポイント種別（ポイント/分岐点/交差点）
- **ポイントID**: ポイント識別子
- **名称**: ポイント名称
- **緯度**: 10進数形式の緯度
- **経度**: 10進数形式の経度

#### 2.1.3 任意列
- **標高**: メートル単位（自動取得可能）
- **備考**: 自由記述欄

#### 2.1.4 データ検証機能
- 必須項目欠け行の自動スキップ
- 座標値範囲チェック（有効な緯度・経度）
- 空行の自動スキップ
- 座標形式の自動変換（DMS形式 → 10進数）

### 2.2 ポイント表示機能

#### 2.2.1 マーカー表示
- **マーカー形式**: CircleMarker（Leaflet.js）
- **マーカー半径**: 6ピクセル（`CONFIG.POINT_MARKER_RADIUS`）
- **マーカー色設定**:
  - **ポイント**: 緑 (#008000)
  - **分岐点**: 薄い紫 (#ba55d3, medium orchid)
  - **交差点**: 濃い茶 (#654321, dark brown)
- **選択状態**: 30%明るい色 + 線の太さ3px + 不透明度0.9

#### 2.2.2 ツールチップ表示
- マーカーホバー時にポイントIDを表示
- 位置: マーカーの上部
- オフセット: [0, -10]ピクセル

#### 2.2.3 ポイント数表示
- **ポイント数**: 区分が「ポイント」または空白のポイント数
- **他の数**: 「分岐点」+「交差点」の合計数
- リアルタイム更新（追加・削除時）

### 2.3 ポイント追加機能

#### 2.3.1 追加モード起動
- **操作**: 「追加」ボタンをクリック
- **ボタン色変化**: 薄い緑 (#90ee90, ライトグリーン)
- **カーソル変化**: 十字カーソル (crosshair)

#### 2.3.2 既存ポイント重複チェック
- **判定方式**: ピクセル距離基準
- **しきい値**: マーカー半径 + 5ピクセル
- **重複時の動作**: 警告メッセージ表示、追加キャンセル
- **カーソルフィードバック**: 既存ポイント付近で「not-allowed」カーソル

#### 2.3.3 新規ポイント追加処理
1. 地図上をクリックしてポイント作成
2. 自動ID生成（区分に応じて）:
   - **ポイント**: 仮01, 仮02, ...
   - **分岐点**: SP01, SP02, ...
   - **交差点**: CP01, CP02, ...
3. GPS標高の自動取得（国土地理院API）
4. ポイントIDフィールドへの自動フォーカス・全選択
5. 追加モード自動解除、ボタン色リセット

### 2.4 ポイント移動機能

#### 2.4.1 移動モード起動
- **前提条件**: ポイントが選択されている
- **操作**: 「移動」ボタンをクリック
- **ボタン色変化**: 薄い緑 (#90ee90, ライトグリーン)
- **カーソル変化**: 移動カーソル (move)

#### 2.4.2 リアルタイム移動処理
- **ドラッグ操作**: マウスダウン → mousemove → マウスアップ
- **リアルタイム座標更新**: ドラッグ中に緯度・経度・DMS表示を動的更新
- **地図ドラッグ無効化**: ポイント移動中は地図移動を一時停止
- **カーソル変化**: ドラッグ中は「grabbing」カーソル

#### 2.4.3 移動完了処理
1. 新座標でポイント位置を更新
2. GPS標高を再取得（国土地理院API）
3. 座標フィールドの最終更新
4. 移動モード自動解除、ボタン色リセット

#### 2.4.4 移動モード解除条件
- ドラッグ完了時
- ESCキー押下時
- 他のポイントをクリック時
- 削除・追加ボタンクリック時

### 2.5 ポイント削除機能

#### 2.5.1 削除処理
- **前提条件**: ポイントが選択されている
- **操作**: 「削除」ボタンをクリック
- **確認ダイアログ**: 「選択したポイント {ID} を削除しますか？」
- **削除実行**: マーカー削除 → データ削除 → 表示クリア → ポイント数更新

### 2.6 ポイント情報編集機能

#### 2.6.1 編集可能フィールド
- **区分**: ドロップダウンリスト（空白/ポイント/分岐点/交差点）
- **ポイントID**: テキスト入力（最大5文字、自動フォーマット機能）
- **名称**: テキスト入力（最大20文字）
- **備考**: テキスト入力（最大50文字）

#### 2.6.2 読み取り専用フィールド
- **緯度**: 10進数形式（小数点以下5桁）
- **経度**: 10進数形式（小数点以下5桁）
- **DMS**: 度分秒形式（東経・北緯順、E/N付き）
- **標高**: メートル単位（小数点1位まで、123.0は123と表示）

#### 2.6.3 区分変更時の自動処理
- **ID自動生成**: 区分変更時に適切なプレフィックス+連番を自動生成
- **マーカー色変更**: 区分に応じた色に自動変更
- **ポイント数更新**: カテゴリ別カウントの再計算

#### 2.6.4 ポイントID自動フォーマット機能
**フォーマットルール**:
1. 全角英数字 → 半角変換、英文字大文字化
2. スペース削除（全角・半角）
3. 末尾1桁数字の場合 → 0パディング（例: "1" → "01"）
4. 3文字以内で英字+数字の場合 → ハイフン挿入（例: "P01" → "P-01"）
5. 日本語文字（漢字・ひらがな・カタカナ）は保持

**処理タイミング**: ポイントIDフィールドのblur（フォーカスアウト）時

#### 2.6.5 タブ順序制御
最適化されたタブ順序:
1. 区分（tabindex="1"）
2. ポイントID（tabindex="2"）
3. 名称（tabindex="3"）
4. 備考（tabindex="4"）

読み取り専用フィールド（緯度・経度・DMS・標高）は`tabindex="-1"`でタブスキップ。

### 2.7 ファイル出力機能

#### 2.7.1 Excel専用出力
- **出力ボタン**: 「ポイント出力(Excel)」
- **ファイル形式**: Excel (.xlsx) のみ
- **デフォルトファイル名**: `ポイントGPS-yyyymmdd.xlsx`
- **保存方式**: File System Access API優先、フォールバック自動ダウンロード

#### 2.7.2 出力データ構造
- **シート名**: 「ポイントGPS」
- **列構成**: 区分、ポイントID、名称、緯度、経度、標高、備考
- **座標精度**: 緯度・経度は小数点以下5桁
- **標高精度**: 小数点1位まで（123.0は123として出力）
- **列幅**: 自動調整（日本語対応、1文字=2幅として計算）

## 3. 技術仕様

### 3.1 技術スタック

#### 3.1.1 フロントエンド
- **言語**: JavaScript ES6 (モジュール構成)
- **地図ライブラリ**: Leaflet.js v1.9.4 (CDN)
- **ファイル処理**: SheetJS v0.18.5 (CDN)
- **スタイリング**: カスタムCSS（レスポンシブ対応）

#### 3.1.2 外部サービス
- **地図タイル**: 国土地理院標準地図タイル
- **標高API**: 国土地理院標高API（5mメッシュ、レーザー測量データ）

#### 3.1.3 ブラウザAPI
- FileReader API（ファイル読み込み）
- File System Access API（ファイル保存）
- Fetch API（標高取得）

### 3.2 モジュール構成

#### 3.2.1 メインアプリケーション
- **PointGPSApp** (`js/app.js`): 全モジュールを統合・管理するメインクラス

#### 3.2.2 コアモジュール
- **MapManager** (`js/map-manager.js`): Leaflet地図の初期化・設定管理
- **GPSDataManager** (`js/gps-data-manager.js`): GPSデータの読み込み・管理・出力
- **PointManager** (`js/point-manager.js`): ポイント表示・編集・移動管理
- **FileHandler** (`js/file-handler.js`): Excelファイル入出力処理

#### 3.2.3 ユーティリティモジュール
- **DataUtils** (`js/data-utils.js`): データ処理・フォーマット・バリデーション
- **ElevationAPI** (`js/elevation-api.js`): 国土地理院標高API連携

#### 3.2.4 設定
- **config.js** (`js/config.js`): アプリケーション全体の設定定数

### 3.3 初期化フロー
1. DOMContentLoaded後にPointGPSAppを初期化
2. MapManagerで地図を初期化
3. FileHandlerを初期化
4. GPSDataManagerを初期化（FileHandlerを注入）
5. PointManagerを初期化（MapManager, GPSDataManagerを注入）
6. イベントハンドラー設定

### 3.4 地図設定
- **初期中心点**: `[34.853667, 135.472041]` (箕面大滝)
- **初期ズームレベル**: 15
- **タイルレイヤー**: `https://cyberjapandata.gsi.go.jp/xyz/std/{z}/{x}/{y}.png`
- **コントロール**:
  - ズーム（左上、右下）
  - スケール（右下、メートル法のみ）

## 4. データ構造

### 4.1 内部ポイントオブジェクト

```javascript
{
  type: string,        // 区分（ポイント/分岐点/交差点）
  id: string,          // ポイントID
  lat: number,         // 緯度 (10進数)
  lng: number,         // 経度 (10進数)
  elevation: string,   // 標高 (API取得値、小数点1位まで)
  location: string,    // 名称（場所名）
  remarks: string      // 備考
}
```

### 4.2 Excel入力形式（厳密仕様）

| 区分   | ポイントID | 名称     | 緯度      | 経度       | 標高 | 備考           |
|--------|------------|----------|-----------|------------|------|----------------|
| ポイント | P-01       | 箕面大滝 | 34.853667 | 135.472041 | 150  | 観光名所       |
| 分岐点  | SP-01      | 分岐1    | 34.854000 | 135.472500 | 155  | 左へ進む       |
| 交差点  | CP-01      | 交差1    | 34.855000 | 135.473000 | 160  | 直進推奨       |

**列名は完全一致必須**:
- 必須: 「区分」「ポイントID」「名称」「緯度」「経度」
- 任意: 「標高」「備考」

### 4.3 Excel出力形式

```
シート名: "ポイントGPS"
列幅: 自動調整（日本語対応）
座標精度: 緯度・経度は小数点以下5桁、標高は小数点1位まで

| 区分   | ポイントID | 名称     | 緯度       | 経度        | 標高 | 備考           |
|--------|------------|----------|-----------|-------------|------|----------------|
| ポイント | P-01       | 箕面大滝 | 34.85367  | 135.47204   | 150  | 観光名所       |
```

## 6. ファイル入出力仕様

- **MIME type**: `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- **拡張子**: `.xlsx`
- **最大行数**: 1000行（`CONFIG.MAX_EXCEL_ROWS`で設定可能）

#### 6.1.2 列検出ロジック
- ヘッダー行（1行目）から列のインデックスを特定
- 完全一致判定（「ポイントID」≠「ID」）
- 必須列の欠落チェック

#### 6.1.3 データ解析処理
1. 2行目以降をデータとして処理
2. 空行の自動スキップ
3. 必須項目欠け行の自動スキップ
4. 座標形式変換（DMS → 10進数）
5. 標高の正規化（小数点1位まで）

#### 6.1.4 エラーハンドリング
- ファイル形式エラー: 「Excelファイル(.xlsx)を選択してください」
- 必須列欠落: 「必須項目が不足しています: {列名リスト}」
- 読み込み失敗: 「Excelファイルの読み込みに失敗しました」

### 6.2 Excel出力仕様

#### 6.2.1 ファイル保存方式
- **優先**: File System Access API（ユーザーが保存場所を選択）
- **フォールバック**: 自動ダウンロード（ブラウザのダウンロードフォルダ）

#### 6.2.2 ワークブック構造
- **シート名**: 「ポイントGPS」
- **ヘッダー行**: 区分、ポイントID、名称、緯度、経度、標高、備考
- **データ行**: 各ポイントの情報

#### 6.2.3 列幅自動調整
- 日本語文字を1文字=2幅として計算
- 最小幅: 8
- 最大幅: 50
- パディング: +2

## 7. 外部API連携

### 7.1 国土地理院標高API

#### 7.1.1 API仕様
- **エンドポイント**: `https://cyberjapandata2.gsi.go.jp/general/dem/scripts/getelevation.php`
- **パラメータ**: `lon={経度}&lat={緯度}&outtype=JSON`
- **データソース**: 5mメッシュ、レーザー測量データ
- **精度**: 小数点1位まで（123.0は123として表示）

#### 7.1.2 取得タイミング（限定的）
- **新規ポイント追加時**: 自動取得
- **ポイント選択時**: 標高がblankまたは0の場合のみ取得
- **ポイント移動完了時**: 強制的に再取得
- **Excel読み込み時**: 取得しない（高速化のため）

#### 7.1.3 取得条件判定
- **blank**: 標高が空文字列または未定義
- **0**: 標高が数値の0
- **それ以外**: 既存値を保持、API呼び出しなし

#### 7.1.4 エラーハンドリング
- API通信エラー: 警告ログ出力、処理継続
- データ取得失敗: 既存値を保持、処理継続

## 12. 制約事項

### 12.1 技術制約
- **CORS制限**: file://プロトコル不可、HTTPサーバー必須
- **座標系**: WGS84のみ対応（日本測地系非対応）
- **ファイル形式**: .xlsxのみ対応（.xls、.csv非対応）
- **GeoJSON**: 非対応（Excel専用化のため削除）

### 12.2 運用制約
- **インターネット接続**: 地図タイル、標高API利用のため必要
- **メモリ制限**: 大量ポイント（1000点以上）では性能劣化の可能性
- **データ永続化**: ブラウザ一時メモリのみ、定期エクスポート推奨

### 12.3 ブラウザ対応
- **推奨**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **制限**: File System Access API未対応ブラウザではダウンロード方式
- **必須**: ES6 Modules, Fetch API, FileReader API
