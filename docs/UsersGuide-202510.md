## 1. はじめに

### 1.1 PointGPSとは

PointGPSは、国土地理院地図上でGPSポイントを視覚的に管理できるWebアプリケーションです。ハイキングや登山のポイント管理、GPS座標データの編集・可視化に最適なツールです。

### 1.2 主な機能

- **Excelファイルの読み込み**: GPSポイントデータをExcelファイル(.xlsx)から読み込み
- **地図上でのポイント表示**: 国土地理院地図上にポイントをマーカーで表示
- **ポイントの種類管理**: ポイント、分岐点、交差点の3種類を色分けして管理
- **ポイントの追加・移動・削除**: マウス操作でポイントを簡単に編集
- **標高の自動取得**: 国土地理院APIからGPS標高を自動取得
- **Excelファイルへの出力**: 編集したデータをExcelファイル(.xlsx)で保存

### 1.3 本手引の対象者

- ハイカー、登山者
- GPS データを扱う方
- 地図アプリケーション利用者

## 2. 動作環境

### 2.1 推奨ブラウザ

- **Google Chrome**: バージョン90以上
- **Mozilla Firefox**: バージョン88以上
- **Microsoft Edge**: バージョン90以上
- **Safari**: バージョン14以上

### 2.2 必要な環境

- **インターネット接続**: 地図表示と標高取得に必要
- **ローカルサーバー**: 開発時のみ（Python、Node.jsなど）
- **JavaScript有効化**: ブラウザでJavaScriptを有効にしてください

### 2.3 対応ファイル形式

- **読み込み**: Excel (.xlsx) のみ
- **出力**: Excel (.xlsx) のみ

## 3. 起動方法

### 3.1 ローカルサーバーでの起動（推奨）

PointGPSはES6モジュールを使用しているため、ローカルサーバー経由でアクセスする必要があります。

#### Pythonを使用する場合

```bash
# プロジェクトフォルダに移動
cd /path/to/PointGPS

# Python 3でHTTPサーバーを起動
python -m http.server 8000
```

#### Node.jsを使用する場合

```bash
# プロジェクトフォルダに移動
cd /path/to/PointGPS

# serveパッケージで起動
npx serve .
```

### 3.2 ブラウザでアクセス

ローカルサーバー起動後、ブラウザで以下のURLにアクセスします：

```
http://localhost:8000
```

## 4. 基本操作

### 4.1 画面構成

PointGPSの画面は以下の要素で構成されています：

1. **地図エリア**: 国土地理院地図が表示されます
2. **コントロールパネル（右上）**: ポイント編集ツール
   - ポイント読み込みボタン
   - ポイント数表示
   - ポイント情報編集エリア
   - ポイント操作ボタン（追加・移動・削除）
   - ポイント出力ボタン
3. **地図コントロール**:
   - ズームボタン（左上、右下）
   - スケール表示（右下）

### 4.2 地図操作

- **拡大・縮小**: マウスホイール、またはズームボタン
- **移動**: 地図をドラッグ
- **ピンチズーム**: タッチデバイスで2本指操作

## 5. ポイントの読み込み

### 5.1 Excelファイルの準備

読み込むExcelファイルは以下の形式で作成してください。

#### 必須列（完全一致）

| 区分   | ポイントID | 名称     | 緯度      | 経度       | 標高 | 備考           |
|--------|------------|----------|-----------|------------|------|----------------|
| ポイント | P-01       | 箕面大滝 | 34.853667 | 135.472041 | 150  | 観光名所       |
| 分岐点  | SP-01      | 分岐1    | 34.854000 | 135.472500 | 155  | 左へ進む       |
| 交差点  | CP-01      | 交差1    | 34.855000 | 135.473000 | 160  | 直進推奨       |

**重要**: 列名は完全一致が必須です。以下の列が必要です：
- **区分**: ポイント種別（ポイント/分岐点/交差点/空白）
- **ポイントID**: ポイントの識別子
- **名称**: ポイント名称
- **緯度**: 10進数形式の緯度
- **経度**: 10進数形式の経度
- **標高**: メートル単位（省略可能、自動取得可能）
- **備考**: 自由記述欄（省略可能）

### 5.2 読み込み手順

1. **「ポイント読み込み(Excel)」ボタンをクリック**
2. **ファイル選択ダイアログでExcelファイルを選択**
3. **読み込み完了を確認**
   - 成功時: 「{N}個のポイントを読み込みました」と表示
   - 失敗時: エラーメッセージを確認

### 5.3 読み込み時の注意点

- **行数制限**: 最大1000行まで読み込み可能
- **データ検証**: 必須項目が欠けている行は自動的にスキップされます
- **座標形式**: 度分秒形式の座標は自動的に10進数に変換されます
- **標高取得**: 読み込み時は標高を自動取得しません（高速化のため）

## 6. ポイントの表示と選択

### 6.1 ポイントの表示

読み込まれたポイントは、区分に応じた色のマーカーで地図上に表示されます：

- **ポイント**: 緑色 (●)
- **分岐点**: 薄い紫色 (●)
- **交差点**: 濃い茶色 (●)

### 6.2 ポイントの選択

ポイントをクリックすると選択状態になり、以下の変化が起こります：

1. **マーカー色が明るくなる**: 選択されたことが視覚的にわかります
2. **ポイント情報が表示される**: 右側のパネルに詳細情報が表示されます
3. **編集可能になる**: ポイントID、名称、備考を編集できます

### 6.3 ポイント数の確認

コントロールパネル上部に、以下の情報が表示されます：

- **ポイント数**: 「ポイント」または空白区分のポイント数
- **他**: 「分岐点」+「交差点」の合計数

## 7. ポイントの追加

### 7.1 追加手順

1. **「追加」ボタンをクリック**
   - ボタンが薄い緑色に変わります
   - カーソルが十字に変わります

2. **地図上の任意の場所をクリック**
   - クリックした位置に新しいポイントが作成されます
   - 自動的にポイントIDが生成されます（仮01, 仮02, ...）

3. **標高が自動取得されます**
   - 国土地理院APIから標高データを取得

4. **ポイントIDフィールドにフォーカス**
   - 自動的にポイントIDフィールドがフォーカスされ、全選択状態になります
   - すぐにIDを編集できます

### 7.2 既存ポイントとの重複防止

- **既存ポイントの近く**: カーソルが禁止マーク (🚫) に変わります
- **重複時の動作**: 「この地点には既にポイントが存在します」と警告が表示され、追加がキャンセルされます

### 7.3 追加モードの解除

以下の操作で追加モードが解除されます：

- ポイントを追加後、自動的に解除
- **ESCキー**を押す
- 他のボタン（移動・削除）をクリック

## 8. ポイントの移動

### 8.1 移動手順

1. **移動したいポイントを選択**
   - 地図上のマーカーをクリック

2. **「移動」ボタンをクリック**
   - ボタンが薄い緑色に変わります
   - カーソルが移動マークに変わります

3. **マーカーをドラッグ**
   - マウスでマーカーを掴み、ドラッグします
   - ドラッグ中、リアルタイムで座標情報が更新されます

4. **マウスボタンを離す**
   - 新しい位置にポイントが移動します
   - 標高が自動的に再取得されます
   - 移動モードが自動的に解除されます

### 8.2 リアルタイム座標表示

ドラッグ中、以下の情報がリアルタイムで更新されます：

- **緯度**: 10進数形式
- **経度**: 10進数形式
- **DMS**: 度分秒形式（東経・北緯順）

### 8.3 移動モードの解除

以下の操作で移動モードが解除されます：

- ドラッグ完了後、自動的に解除
- **ESCキー**を押す
- 他のポイントをクリック
- 他のボタン（追加・削除）をクリック

## 9. ポイントの編集

### 9.1 編集可能な項目

選択されたポイントの以下の項目を編集できます：

#### 9.1.1 区分（ドロップダウンリスト）

ポイントの種類を選択します：

- **空白**: 区分なし
- **ポイント**: 一般的なポイント（緑色マーカー）
- **分岐点**: 道の分岐点（薄い紫色マーカー）
- **交差点**: 道の交差点（濃い茶色マーカー）

**区分変更時の自動処理**:
- ポイントIDが自動生成されます（例: 分岐点 → SP01）
- マーカー色が変更されます
- ポイント数カウントが更新されます

#### 9.1.2 ポイントID（テキスト入力、最大5文字）

ポイントの識別子を入力します。

**自動フォーマット機能**:
- フォーカスアウト時に自動的にフォーマットされます
- 全角英数字 → 半角変換
- 英文字 → 大文字化
- 1桁数字 → 2桁にゼロパディング（例: "1" → "01"）
- 3文字以内の英字+数字 → ハイフン挿入（例: "P01" → "P-01"）
- 日本語文字（漢字・ひらがな・カタカナ）は保持

**入力例**:
- `p1` → `P-01`
- `sp2` → `SP-02`
- `仮1` → `仮01`

#### 9.1.3 名称（テキスト入力、最大20文字）

ポイントの名称を入力します（例: 箕面大滝、分岐1）。

#### 9.1.4 備考（テキスト入力、最大50文字）

ポイントに関する補足情報を自由に入力します。

### 9.2 読み取り専用項目

以下の項目は自動計算されるため、編集できません：

- **緯度**: 10進数形式（小数点以下5桁）
- **経度**: 10進数形式（小数点以下5桁）
- **DMS**: 度分秒形式（東経・北緯順、E/N付き）
- **標高**: メートル単位（小数点1位まで、123.0は123と表示）

### 9.3 タブキー操作

効率的な編集のため、タブキーで以下の順序で移動します：

1. 区分
2. ポイントID
3. 名称
4. 備考

読み取り専用フィールド（緯度・経度・DMS・標高）はタブキーでスキップされます。

## 10. ポイントの削除

### 10.1 削除手順

1. **削除したいポイントを選択**
   - 地図上のマーカーをクリック

2. **「削除」ボタンをクリック**
   - 確認ダイアログが表示されます

3. **確認ダイアログで「OK」をクリック**
   - ポイントが削除されます
   - 「ポイント {ID} を削除しました」とメッセージが表示されます

### 10.2 削除のキャンセル

確認ダイアログで「キャンセル」をクリックすると、削除がキャンセルされます。

## 11. ポイントの出力

### 11.1 出力手順

1. **「ポイント出力(Excel)」ボタンをクリック**
   - ファイル保存ダイアログが表示されます（ブラウザによって異なります）

2. **ファイル名と保存場所を指定**
   - デフォルトファイル名: `ポイントGPS-yyyymmdd.xlsx`
   - 任意のファイル名に変更可能

3. **保存完了を確認**
   - 「Excelファイルを保存しました: {ファイル名}」と表示されます

### 11.2 出力ファイルの形式

出力されるExcelファイルは以下の形式です：

- **シート名**: 「ポイントGPS」
- **列構成**: 区分、ポイントID、名称、緯度、経度、標高、備考
- **座標精度**: 緯度・経度は小数点以下5桁
- **標高精度**: 小数点1位まで（123.0は123として出力）
- **列幅**: 日本語に対応して自動調整

### 11.3 ブラウザ別の保存動作

- **Chrome/Edge（File System Access API対応）**: 保存場所を選択できます
- **その他のブラウザ**: ダウンロードフォルダに自動保存されます

## 12. トラブルシューティング

### 12.1 ファイルが読み込めない

#### 問題: 「Excelファイル(.xlsx)を選択してください」と表示される

**原因**: ファイル形式が正しくない

**解決方法**:
- Excel 2007以降の.xlsx形式で保存されているか確認
- .xls（旧形式）や.csv形式は非対応です

#### 問題: 「必須項目が不足しています」と表示される

**原因**: 必須列が欠けている

**解決方法**:
- 1行目（ヘッダー行）に以下の列があるか確認:
  - 区分
  - ポイントID
  - 名称
  - 緯度
  - 経度
- 列名は完全一致が必要（例: 「ID」ではなく「ポイントID」）

### 12.2 地図が表示されない

#### 問題: 地図タイルが表示されない

**原因**: インターネット接続の問題

**解決方法**:
- インターネット接続を確認
- ブラウザのコンソールエラーを確認（F12キー）
- ファイアウォールやプロキシ設定を確認

#### 問題: 「CORS エラー」が表示される

**原因**: ローカルサーバーを使用していない

**解決方法**:
- ブラウザで直接HTMLファイルを開くのではなく、ローカルサーバー経由でアクセス
- Pythonまたはnpx serveでサーバーを起動

### 12.3 標高が取得できない

#### 問題: 標高フィールドが空白のまま

**原因**:
- 国土地理院API接続エラー
- 対象地点がAPI対応範囲外

**解決方法**:
- インターネット接続を確認
- ブラウザのコンソールで警告メッセージを確認
- 対象地点が日本国内か確認

### 12.4 ポイントが追加できない

#### 問題: 「この地点には既にポイントが存在します」と表示される

**原因**: クリック位置が既存ポイントに近すぎる

**解決方法**:
- 既存ポイントから離れた場所をクリック
- 地図を拡大して、より正確な位置をクリック

### 12.5 ファイルが保存できない

#### 問題: ファイル保存ダイアログが表示されない

**原因**: ブラウザのポップアップブロック

**解決方法**:
- ブラウザのポップアップブロックを解除
- ダウンロードフォルダを確認（自動ダウンロードされている場合があります）

### 12.6 その他の問題

#### キーボード操作が効かない

**原因**: フォーカスが正しくない

**解決方法**:
- 一度入力フィールドをクリックしてフォーカスを当てる
- ページをリロード（F5キー）

#### メッセージが消えない

**原因**: 表示時間が長い（エラーメッセージは6秒表示）

**解決方法**:
- 自動的に消えるまで待つ
- ページをリロード（F5キー）

## 付録B: よくある質問（FAQ）

### Q1: GeoJSON形式は対応していますか？

**A**: Version 2.0ではExcel専用化により、GeoJSON形式は非対応です。Excel (.xlsx) のみ対応しています。

### Q2: 複数のExcelファイルを同時に読み込めますか？

**A**: 現在は1ファイルずつの読み込みのみ対応しています。複数ファイルを読み込むと、前のデータが上書きされます。

### Q3: オフラインで使用できますか？

**A**: 地図タイルと標高APIの取得にインターネット接続が必要なため、完全なオフライン使用はできません。

### Q4: 最大何個のポイントを扱えますか？

**A**: 読み込み時は最大1000ポイントまで対応しています。それ以上のポイント数では性能が劣化する可能性があります。

### Q5: 座標系は何を使用していますか？

**A**: WGS84（世界測地系）のみ対応しています。日本測地系は非対応です。

### Q6: スマートフォンやタブレットで使用できますか？

**A**: はい、レスポンシブデザインに対応しており、タッチ操作も可能です。ただし、画面サイズによっては操作性が制限される場合があります。
